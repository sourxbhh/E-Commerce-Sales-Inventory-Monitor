# FakeStoreAPI integration

The project pulls its product catalog live from
[fakestoreapi.com](https://fakestoreapi.com) — a free, no-auth public
API with 20 real products across 4 categories (electronics, jewelery,
men's clothing, women's clothing). A FastAPI service sits at the edge
of the pipeline, hydrating the catalog, mining basket patterns, and
generating live order traffic that Kafka + Spark consume downstream.

## Why FakeStoreAPI

- **No API key.** Zero friction.
- **Real product data** (titles, prices, categories, ratings, images).
- **Real basket patterns** via `/carts` — we mine basket size and
  quantity distributions from 7 historical carts to make generated
  orders statistically realistic.
- **Free, unlimited**, at a demo-scale rate.

## What we synthesize (because FakeStoreAPI doesn't expose it)

| Field | Source |
|---|---|
| `stock_quantity` | `80 + sqrt(rating.count) * 12`, capped at 600 |
| `reorder_threshold` | 20% of stock, floored at 10 |
| `popularity_weight` | `1 + log10(rating.count + 1)`, capped at ~4.5 |
| order / customer_id / timestamps | generated by the FastAPI service |

So product identity and prices are real; inventory and transactions
are synthetic but shaped by real rating signals (popular items get
stocked heavier AND picked more often).

## Components

```
fakestoreapi.com
    │
    │  (GET /products, GET /carts on startup + on resync)
    ▼
scripts/fakestore_sync.py   ──►  MySQL.products
    │                                 │
    │                                 ▼
    │                         api/main.py  (FastAPI)
    │                                 │
    │        ┌────────────────────────┼────────────────────────┐
    │        ▼                        ▼                        ▼
    │  WebSocket clients       Kafka orders topic        HTTP /products
    │  (ws://…/stream/orders)  (downstream: Spark)       (HTTP consumers)
```

Files:

- [`fakestore/client.py`](../fakestore/client.py) — sync + async httpx client.
- [`scripts/fakestore_sync.py`](../scripts/fakestore_sync.py) — one-shot catalog ingest.
- [`api/order_engine.py`](../api/order_engine.py) — real-catalog order builder.
- [`api/main.py`](../api/main.py) — FastAPI app, WebSocket fan-out, Kafka publish, demo triggers.

## FastAPI endpoints

| Method | Path | Purpose |
|---|---|---|
| GET | `/health` | liveness + orders_sent counter |
| GET | `/stats` | rate, uptime, subscribers, drain state |
| GET | `/products` | hydrated catalog |
| GET | `/products/{id}` | single product |
| POST | `/trigger/burst?size=N` | inject a flash-sale burst |
| POST | `/trigger/drain?product_id=N&times=M` | deplete one SKU |
| WS | `/stream/orders` | live JSON orders as they publish |
| GET | `/docs` | Swagger UI |

## Quick usage

```bash
# one-time bootstrap (fetches real catalog into MySQL)
python scripts/fakestore_sync.py

# start everything (orchestrator runs the sync too)
python scripts/orchestrator.py up

# watch orders stream in real-time over WebSocket
python -c "
import asyncio, websockets, json
async def run():
    async with websockets.connect('ws://localhost:8000/stream/orders') as ws:
        for _ in range(10):
            print(json.loads(await ws.recv())['total_amount'])
asyncio.run(run())
"

# inspect catalog
python scripts/demo_controller.py list-products

# trigger a burst from the CLI (which hits POST /trigger/burst)
python scripts/demo_controller.py burst --size 60

# drain a specific SKU to low-stock
python scripts/demo_controller.py drain --product 1 --times 40

# refresh catalog from FakeStoreAPI
python scripts/orchestrator.py resync
```

## Refresh cadence

FakeStoreAPI data is static in practice — titles/prices don't change
between calls. We still re-sync on each `orchestrator up` so:

1. New deploys always start with a known-good catalog.
2. If the upstream does change, we pick it up on the next boot or
   explicit `orchestrator resync`.

For a truly live signal you'd want an API like Best Buy or Etsy
(noted in `docs/CLOUD_DEPLOYMENT.md`). FakeStoreAPI is the zero-auth,
zero-cost trade.

## Troubleshooting

- **`products table empty after sync`** → FakeStoreAPI may have
  transiently 503'd. Re-run `python scripts/fakestore_sync.py` and
  check `logs/api.log`.
- **WebSocket connects but no messages** → look at
  `logs/api.log` — typically means no products loaded, or the
  generator loop is in a burst pause.
- **Kafka delivery errors in `logs/api.log`** → check Kafka is up:
  `docker compose -f docker/docker-compose.yml ps`.
- **`Duplicate entry for key products.PRIMARY`** on resync → safe to
  ignore; upsert handles it on the next row.
